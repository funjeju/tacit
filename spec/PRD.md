# PRD.md — Tacit Product Requirements Document

## 1. 제품 비전

### 1.1 한 문장 비전
> "한국의 4-50대 도메인 보유자가 AI를 자기 자산으로 바꾸는 유일한 플랫폼"

### 1.2 미션
중년층의 30년 경험은 사회의 가장 저평가된 자산이다. Tacit은 이 암묵지를 AI가 질문으로 끌어내 형식지(프롬프트·지식 자산)로 변환하고, 그 자산이 쓸수록 복리로 쌓이게 한다.

### 1.3 북극성 지표 (North Star Metric)
**월간 활성 사용자당 저장된 프롬프트·프로필 자산 수**
- 사용자가 "돌아올 이유"는 자기 자산이 쌓여 있기 때문
- 단순 MAU가 아닌 "자산화된 MAU"가 본질

---

## 2. 문제 정의

### 2.1 4-50대가 겪는 AI 활용 장벽
1. **입력 장벽**: "뭘 물어야 할지 모르겠다"
2. **출력 해석 장벽**: "답은 받았는데 이게 좋은 답인지 모르겠다"
3. **재사용 장벽**: "잘 쓴 프롬프트를 어디 저장해야 할지 모르겠다"
4. **도구 파편화**: ChatGPT, Claude, Midjourney, Gamma 각각 따로 써야 함

### 2.2 기존 해결책의 한계
- **ChatGPT**: 빈 입력창만 있음 → 초보자는 뭘 쓸지 모름
- **PromptBase**: 전문가용 마켓 → 일반인은 거래 주체가 될 수 없음
- **FlowGPT**: 커뮤니티 기반 → 한국어/한국 맥락 부족
- **AutoGPT 등**: 너무 기술적 → 중년층 진입 불가

### 2.3 Tacit의 해결 방식
"빈 화면" 대신 **"유형 선택 → 질문 응답"** 인터페이스를 제공한다. 사용자는 자기 경험을 말하기만 하면, AI가 프롬프트와 결과물까지 조립한다.

## 3. 타깃 사용자 상세

### 3.1 페르소나 #1: 김영호 (55세, 식당 사장)
- **배경**: 서울 마포에서 20년 한식당 운영
- **상황**: 인스타그램 홍보 필요하지만 사진·글 작성 막막
- **니즈**: 신메뉴 사진 멋지게 + 인스타 문구 + 손님 리뷰 대응 멘트
- **Tacit 사용 시나리오**: "식당 사장 템플릿" 선택 → "신메뉴 홍보" 유형 → 재료·가격·타깃 고객 답변 → 포스터 이미지 + 인스타 문구 자동 생성

### 3.2 페르소나 #2: 박미경 (48세, 초등교사)
- **배경**: 공립 초등학교 25년 근무
- **상황**: 수업 자료 준비 부담, 학부모 소통 민감
- **니즈**: 학년별 수업 자료, 학부모 알림장, 평가 문항
- **Tacit 사용 시나리오**: "교사 템플릿" → "학부모 알림장" → 상황 답변 → 맥락 맞는 알림장 초안 생성

### 3.3 페르소나 #3: 이종석 (52세, 공인중개사)
- **배경**: 제주 서귀포 15년차 공인중개사
- **상황**: 매물 설명글 작성에 매번 시간 투자
- **니즈**: 매물 사진에 어울리는 설명, 지역 상권 분석, 고객 문의 응대
- **Tacit 사용 시나리오**: "공인중개사 템플릿" → "매물 설명글" → 매물 정보 + 타깃 고객 답변 → 전문가 수준 설명문 생성

## 4. 핵심 가치 제안

### 4.1 사용자 관점
- **"프롬프트 엔지니어링 몰라도 전문가 수준 결과물"**
- **"쓸수록 나만의 프로필이 쌓여 결과물이 점점 좋아진다"**
- **"20-30년 경험이 디지털 자산이 된다"**

### 4.2 플랫폼 관점
- **암묵지 데이터 축적**: 각 도메인의 고수들이 자발적으로 자기 지식을 입력
- **네트워크 효과**: Square를 통해 프롬프트/프로필이 공유·거래되며 가치 증폭
- **대체 불가 해자**: 도메인별 질문 트리와 누적된 사용자 프로필은 복제 불가

## 5. 기능 요구사항

### 5.1 Layer 1: 메타프롬프트 엔진 (MVP 필수)

#### 기능
- 사용자가 산출물 유형 선택 (보고서/이미지/영상/PPT/코드/음악/기획서/광고카피)
- AI가 유형에 맞는 질문 5-7개를 동적 생성
- 사용자 답변 기반으로 최적화된 프롬프트 출력

#### 수용 기준
- 모든 유형에서 평균 5분 이내 프롬프트 완성
- 생성된 프롬프트를 ChatGPT/Claude/Midjourney 등 외부 도구에서 바로 사용 가능
- 답변 중단 시에도 자동 저장되어 재개 가능

### 5.2 Layer 2: 원클릭 실행 & 서재 (MVP 필수)

#### 기능
- 프롬프트 생성 완료 시 "[도구명]에서 열기" 버튼 제공
- 버튼 클릭 시 프롬프트 클립보드 복사 + 해당 도구 신규 탭 오픈
- API 연동 가능한 도구(Claude, OpenAI)는 플랫폼 내에서 직접 실행
- 결과물 자동 저장 ("내 서재")

#### 수용 기준
- 지원 도구 MVP: ChatGPT, Claude, Midjourney, DALL-E, Gamma, Veo, Suno, Runway
- 서재 내 결과물 재편집 가능
- 결과물별 "원본 프롬프트 보기" 지원

### 5.3 Layer 3: 도메인별 특화 템플릿 (MVP 필수 — 해자의 핵심)

#### 기능
- 일반 메타프롬프트 위에 도메인 레이어
- MVP 도메인 3종: 식당 사장, 교사, 공인중개사
- 각 도메인별 10-15개 하위 유형 (예: 식당 → 신메뉴, 매장홍보, 리뷰대응, 메뉴판, 스토리텔링 등)

#### 수용 기준
- 도메인 선택 시 하위 유형이 직관적으로 나열
- 도메인별 전문 용어 반영 (예: 교사 → "학습지" 용어 사용, "콘텐츠" 아님)
- 도메인별 톤앤매너 반영

### 5.4 Layer 4: 암묵지 인터뷰 모드 ⭐ (2차 필수 — 킬러 기능)

#### 기능
- "당신의 30년을 인터뷰해드립니다" 모드
- AI가 30-50개 질문으로 사용자의 업무 방식·판단 기준·노하우 추출
- 결과물을 "도메인 프로필"로 저장
- 이후 모든 프롬프트 생성 시 프로필 자동 반영

#### 수용 기준
- 인터뷰 세션 1회당 20-40분 소요
- 중단/재개 자유
- 음성 입력 옵션 (Web Speech API)
- 프로필은 JSON + 텍스트 요약 이중 저장

### 5.5 Layer 5: Square 네트워크 (3차)

#### 기능
- 사용자가 만든 프롬프트 공개/비공개 설정
- 공개 프롬프트는 Square에서 검색·복제·좋아요 가능
- 도메인별 랭킹
- 유료 프롬프트 거래 (장기 로드맵)

#### 수용 기준
- 프롬프트 복제 시 원작자 자동 크레딧
- 좋아요·사용수·평점 기반 랭킹
- 스팸/부적절 프롬프트 신고 기능

## 6. 비기능 요구사항

### 6.1 성능
- LCP ≤ 2.5초
- Claude API 응답 스트리밍 (체감 속도 향상)
- 이미지 lazy loading

### 6.2 접근성
- WCAG 2.1 AA 준수
- 기본 폰트 크기 16px 이상 (4-50대 가독성)
- 터치 타깃 44x44px 이상
- 스크린리더 지원

### 6.3 국제화
- 1차: 한국어
- 2차: 영어, 일본어 (Tacit 네이밍이 글로벌 친화적이므로 확장성 고려)

### 6.4 보안
- 모든 API 키 서버 사이드 전용
- Firebase 보안 규칙 엄격 (기본 deny)
- 개인 프로필 데이터 암호화 저장
- GDPR/개인정보보호법 대응

### 6.5 디바이스 지원
- 모바일 우선 (iPhone 12+, Android 10+)
- 태블릿 최적화
- 데스크톱 (Chrome, Safari, Edge, Firefox 최신 2버전)

## 7. 성공 지표 (KPI)

### 7.1 MVP 런칭 후 3개월
- 회원가입 5,000명
- MAU 2,000명
- 저장된 프롬프트 10,000개
- 완료된 암묵지 인터뷰 500건

### 7.2 6개월
- MAU 10,000명
- 유료 전환율 3%
- Square 공유 프롬프트 3,000개

### 7.3 12개월
- MAU 50,000명
- 유료 사용자 2,000명
- B2B 계약 10건 (소상공인협회 등)

## 8. 범위 외 (Out of Scope)

MVP에서 제외:
- 다국어 지원 (한국어 외)
- 모바일 네이티브 앱 (PWA로 대응)
- 오프라인 모드
- 팀 협업 기능 (2차)
- 자동 결제 (초기는 수동 구독)

## 9. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| 4-50대 UX 난이도 | 매주 5명 사용성 테스트, 초기 피드백 반영 |
