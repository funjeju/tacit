# USER_FLOWS.md — 사용자 플로우

Tacit의 핵심 사용자 여정을 도식화한 문서입니다. 모든 플로우는 4-50대 사용자가 **중간에 멈추지 않고 완주**할 수 있도록 설계되었습니다.

---

## 원칙

1. **3클릭 이내 핵심 가치 도달** — 랜딩 → 유형 선택 → 첫 질문
2. **언제든 이전 단계로 복귀 가능** — 모든 화면에 "뒤로", "저장하고 나가기" 제공
3. **저장 강박 제거** — 모든 입력은 자동 저장 (ephemeral 아님)
4. **실수 용납** — 삭제·변경은 항상 확인 다이얼로그 + 되돌리기

---

## Flow 1. 최초 온보딩 (비로그인 → 첫 프롬프트 완성)

### 목표
비회원 사용자가 **회원가입 없이도** 첫 메타프롬프트를 체험하게 한다. 가입은 결과 저장 시점에 유도한다.

### 단계

```
[1] 랜딩페이지 (/)
    ├─ 헤드라인: "당신의 30년 경험을, AI가 대신 묻습니다"
    ├─ CTA 버튼: "지금 체험하기 (회원가입 없이)"
    └─ 하단: 산출물 유형 6개 아이콘 (보고서·이미지·영상·PPT·코드·음악)

        ↓ 클릭

[2] 산출물 유형 선택 (/create)
    ├─ 6개 큰 카드 (각 120x120px 이상)
    ├─ 각 카드에 "이런 걸 만들 수 있어요" 예시 3개
    └─ 선택 시 즉시 다음 단계로 (확인 버튼 없음)

        ↓ 선택 (예: "이미지")

[3] 초기 키워드 입력 (/create/[type]/seed)
    ├─ 큰 텍스트 영역 (16px 이상, placeholder 예시 있음)
    ├─ "말로 입력하기" 마이크 버튼 (STT)
    ├─ 음성 도움말: "뭘 만들고 싶으신지 편하게 입력하세요"
    └─ [다음] 버튼

        ↓ 입력 (예: "제주 카페 신메뉴 포스터")

[4] 질문 인터뷰 세션 (/create/[type]/interview)
    ├─ 하단 고정 InterviewBar: "질문 3/7"
    ├─ 한 번에 질문 1개만 표시 (카드 형식)
    ├─ 답변 입력 영역 (텍스트 + 음성)
    ├─ "이 질문 건너뛰기" 버튼
    ├─ "잘 모르겠어요, 추천해주세요" 버튼 → AI가 기본값 제시
    └─ 모든 질문 완료 시 → 자동 [5]로 이동

        ↓ 7개 질문 완료

[5] 프롬프트 생성 중 로딩 (/create/[type]/generating)
    ├─ 프로그레스: "당신의 답변을 프롬프트로 번역 중"
    ├─ SSE 스트리밍으로 한 글자씩 나타남
    └─ 평균 5~12초

        ↓ 완료

[6] 결과 화면 (/create/[type]/result/[id])
    ├─ 완성된 프롬프트 (복사 가능, 편집 가능)
    ├─ 대형 버튼: "[Midjourney]에서 열기" (유형별)
    ├─ 보조 버튼: "프롬프트 복사", "저장하기"
    └─ "저장하기" 클릭 시 → [7] 가입 유도

        ↓ "저장하기" 클릭 (비회원)

[7] 가입 유도 모달 (/auth/signup?returnTo=...)
    ├─ 헤드라인: "방금 만든 프롬프트를 저장하려면 가입이 필요해요"
    ├─ 소셜 가입 3종 (Google, Kakao, Naver) - 원탭
    ├─ 또는 "이메일로 가입" (이메일+비밀번호 2필드)
    └─ 가입 완료 시 → 자동으로 [6] 결과가 "내 서재"에 저장됨

        ↓ 가입 완료

[8] 첫 성공 경험 (/library)
    ├─ "첫 프롬프트가 서재에 저장됐어요!" 토스트
    ├─ 온보딩 투어 시작 (OnboardingTour 컴포넌트)
    │   ├─ Step 1: "여기가 당신의 서재입니다"
    │   ├─ Step 2: "이 버튼을 누르면 언제든 다시 쓸 수 있어요"
    │   └─ Step 3: "다음엔 '암묵지 인터뷰'로 당신만의 AI를 만들어보세요"
    └─ CTA: "암묵지 인터뷰 시작하기"
```

### 성공 지표
- 랜딩 → [6] 결과 도달률 ≥ 30%
- [6] → [8] 가입 전환율 ≥ 50%
- 평균 완주 시간 ≤ 7분

## Flow 2. 암묵지 인터뷰 (킬러 플로우)

### 목표
사용자의 **도메인 경험·판단 기준·노하우**를 30~50개 질문으로 구조화하여 `DomainProfile`로 저장. 이후 모든 메타프롬프트에 자동 반영된다.

### 단계

```
[1] 인터뷰 진입 (/interview)
    ├─ 헤드라인: "당신의 경험을 들려주세요"
    ├─ 설명: "AI가 30~50개 질문으로 당신만의 노하우를 정리합니다"
    ├─ 예상 소요 시간: "20~40분 (중간 저장 가능)"
    └─ [시작하기] 버튼

        ↓

[2] 기본 정보 (/interview/basic)
    ├─ 질문 1: "어떤 분야에서 일하시나요?" (도메인 선택)
    │   └─ 6개 MVP 도메인 + "기타 직접 입력"
    ├─ 질문 2: "그 분야에서 몇 년째 일하셨나요?"
    │   └─ 슬라이더 (0~50년)
    ├─ 질문 3: "주로 어떤 일을 하시나요?" (하위 유형)
    │   └─ 선택한 도메인의 하위 유형 목록
    └─ [다음] 버튼

        ↓

[3] 심층 인터뷰 (/interview/deep)
    ├─ 6개 카테고리 순차 진행:
    │   1. 일상 업무 (5~8개 질문)
    │   2. 판단 기준 (5~8개 질문)
    │   3. 성공 경험 (5~8개 질문)
    │   4. 실패 경험 (5~8개 질문)
    │   5. 고객/상대 이해 (5~8개 질문)
    │   6. 노하우·팁 (5~8개 질문)
    │
    ├─ 각 질문마다:
    │   ├─ 텍스트 입력 + 음성 입력 + 건너뛰기
    │   ├─ 하단 프로그레스: "28/45"
    │   ├─ 카테고리 변경 시 축하 애니메이션 ("일상 업무 완료!")
    │   └─ 3분마다 자동 저장 (Interview 컬렉션)
    │
    └─ 45개 질문 완료 시 → [4]

        ↓

[4] 프로필 생성 중 (/interview/generating)
    ├─ 프로그레스: "답변 45개를 분석해 프로필을 만드는 중"
    ├─ Claude API 호출 (Sonnet 4.6, 전체 답변 → 구조화된 프로필)
    ├─ 평균 30~60초
    └─ 완료 시 자동 [5]로

        ↓

[5] 프로필 미리보기 (/interview/preview)
    ├─ 생성된 프로필 표시:
    │   ├─ 요약 1줄: "25년차 제주 해산물 전문 식당 운영자"
    │   ├─ 주요 강점 5개
    │   ├─ 판단 기준 5개
    │   ├─ 전문 어휘 20개
    │   └─ 커뮤니케이션 스타일
    ├─ "이 부분은 다르게 수정" 인라인 편집 가능
    ├─ [저장하고 활성화] 버튼
    └─ 저장 시 Supabase pgvector에 임베딩도 함께 저장

        ↓

[6] 활성화 완료 (/profile/[id])
    ├─ "프로필이 활성화됐습니다" 축하 토스트
    ├─ 이후 모든 프롬프트 생성 시 이 프로필이 자동 반영
    └─ CTA: "이 프로필로 첫 프롬프트 만들기" → Flow 1의 [2]로
```

### 중단·재개
- 언제든 `[나가기]` 버튼으로 종료 가능
- `/interview/resume`에서 이어하기 자동 제안
- 24시간 이상 중단 시 "이어하시겠어요?" 이메일 리마인더

### 성공 지표
- 인터뷰 시작 → 완료율 ≥ 40%
- 평균 완주 시간 ≤ 35분
- 프로필 활성화 후 30일 내 프롬프트 생성 ≥ 5회

## Flow 3. 메타프롬프트 생성 (로그인 사용자)

### 목표
로그인된 사용자가 **기존 프로필 + 도메인 템플릿**을 활용해 빠르게 프롬프트를 생성한다.

### 단계

```
[1] 홈 대시보드 (/dashboard)
    ├─ 상단: "최근 프롬프트" 가로 스크롤
    ├─ 중단: [+ 새로 만들기] 큰 버튼
    └─ 하단: "내 프로필로 추천하는 프롬프트" 3개

        ↓ [+ 새로 만들기]

[2] 유형 선택 (/create)
    ├─ 프로필이 있으면: "당신의 프로필 '제주 식당 사장' 기준으로" 배지
    └─ 6개 유형 카드

        ↓

[3] 도메인 템플릿 선택 (/create/[type]/template) - 선택적
    ├─ 프로필이 있으면: 해당 도메인의 템플릿 자동 필터링
    ├─ 예: 식당 사장 × 이미지 → "신메뉴 포스터", "매장 홍보", "메뉴판 리뉴얼"
    ├─ 또는 [처음부터 만들기] 선택
    └─ 템플릿 선택 시 → 질문 인터뷰의 질문이 자동 세팅됨

        ↓

[4] 질문 인터뷰 (/create/[type]/interview) — Flow 1의 [4]와 동일
    └─ 단, 프로필에서 추론 가능한 답변은 "추천 답변" 으로 미리 채워짐

        ↓

[5] 결과 (/create/[type]/result/[id])
    ├─ 프롬프트 자동 저장됨 (가입 유도 없음)
    ├─ 3초 이내 응답
    └─ Square 공유 버튼 표시 (선택)
```

## Flow 4. Square 탐색 및 복제

### 목표
타인이 만든 고품질 프롬프트를 발견하고, **내 서재로 복제**하여 수정·실행한다.

### 단계

```
[1] Square 진입 (/square)
    ├─ 상단 탭: "추천" | "최신" | "인기" | "내 도메인"
    │   └─ "내 도메인" = 프로필 기반 벡터 유사도 추천
    ├─ 카드 그리드 (SquareCard):
    │   ├─ 프롬프트 제목
    │   ├─ 작성자 닉네임 + 도메인 배지
    │   ├─ 좋아요 수, 복제 수
    │   └─ 샘플 결과물 썸네일 (있으면)
    └─ 무한 스크롤

        ↓ 카드 클릭

[2] Square 상세 (/square/[id])
    ├─ 전체 프롬프트 (복사 가능, 편집 불가)
    ├─ 작성자 프로필 요약
    ├─ 사용 예시 결과물 (있으면)
    ├─ 좋아요·신고 버튼
    └─ [내 서재로 복제하기] 큰 버튼

        ↓ 복제

[3] 복제 완료 (/library/[id])
    ├─ 내 서재에 복사본 저장됨 (원본과 링크됨)
    ├─ "복제했습니다" 토스트
    ├─ 출처 표시: "원본: @작성자님"
    └─ 바로 편집·실행 가능
```

### 유료 프롬프트 (2차)
```
[2'] 상세 → [결제하고 잠금 해제] 버튼
      ↓
[3'] 결제 모달 (Stripe or 토스페이)
      ↓
[4'] 구매 완료 후 [3]과 동일한 복제 플로우
```

## Flow 5. 프로필 편집

### 목표
기존 `DomainProfile`을 **일부만** 수정하여 최신 상태로 유지한다. 전체 재인터뷰는 요구하지 않는다.

### 단계

```
[1] 프로필 목록 (/profile)
    ├─ 활성 프로필 1개 + 비활성 프로필들
    └─ 각 프로필 카드

        ↓ 편집

[2] 프로필 상세 (/profile/[id])
    ├─ 섹션별 표시:
    │   ├─ 기본 정보 (도메인, 경력)
    │   ├─ 강점 5개
    │   ├─ 판단 기준 5개
    │   ├─ 전문 어휘
    │   └─ 커뮤니케이션 스타일
    ├─ 각 섹션 옆 [편집] 버튼
    ├─ [활성화 / 비활성화] 토글
    └─ [추가 인터뷰로 보강하기] 버튼 → Flow 2의 특정 카테고리만 재실행

        ↓ 인라인 편집

[3] 자동 저장 + 임베딩 재생성
    └─ 변경 사항 저장 시 Supabase 임베딩 자동 업데이트
```

## Flow 6. 구독 전환 (Freemium → Pro)

### 목표
무료 사용량 초과 시점에 **자연스럽게** Pro로 유도한다.

### 단계

```
[1] 사용량 한도 임박 (9/10 사용 시)
    ├─ 상단 배너: "이번 달 무료 사용량 1회 남았어요"
    └─ [Pro 둘러보기] 버튼 (비강제)

        ↓ 10회 전부 소진

[2] 다음 프롬프트 시도 시 모달 (/upgrade)
    ├─ 헤드라인: "이번 달 무료 사용량을 다 쓰셨어요"
    ├─ 설명: "Pro로 업그레이드하면 제한 없이 사용 가능합니다"
    ├─ 플랜 비교표:
    │   ├─ Free: 월 10회, 서재 50개 제한
    │   └─ Pro ₩9,900/월: 무제한, 서재 무제한, 고급 템플릿
    ├─ [Pro 시작하기] 버튼
    └─ [다음 달까지 기다리기] 보조 버튼

        ↓ Pro 시작

[3] 결제 (/billing/checkout)
    ├─ 토스페이먼츠 연동 (카드, 간편결제)
    ├─ 첫 7일 무료 체험 (옵션)
    └─ 결제 완료 시 → 즉시 Pro 권한 활성화

        ↓

[4] 완료 (/billing/success)
    ├─ "Pro 사용자가 되셨습니다" 축하 모달
    ├─ Pro 전용 기능 3가지 소개
    └─ 원래 만들려던 프롬프트 화면으로 복귀
```

### 이탈 방지
- 결제 화면에서 이탈 시 24시간 후 "10% 할인 쿠폰" 이메일

## 플로우 간 연결 다이어그램

```
랜딩 ──→ 체험(Flow 1) ──→ 가입 ──→ 대시보드
                                    │
                  ┌─────────────────┼─────────────────┐
                  ↓                 ↓                 ↓
            암묵지 인터뷰       프롬프트 생성       Square 탐색
            (Flow 2)           (Flow 3)           (Flow 4)
                  │                 │                 │
                  └──→ 프로필 활성화 ←── 복제 ←───────┘
                                    │
                                    ↓
                              월 10회 초과
                                    │
                                    ↓
                             구독 전환 (Flow 6)
```

## 에러 복구 플로우

### API 실패 시
```
[프롬프트 생성 실패]
    ├─ 재시도 버튼 (최대 3회 자동 재시도)
    ├─ "잠시 후 다시 시도해주세요" 메시지
    └─ 입력 내용은 자동 저장되어 있음 (사용자가 다시 입력할 필요 없음)
```

### 세션 만료
```
[401 감지]
    ├─ 작업 중인 내용 localStorage에 임시 저장
    ├─ "/auth/login?returnTo=..." 으로 리다이렉트
    └─ 재로그인 후 자동 복구
```
